# 31.01.15 Capabilities

Разделяют привилегии root на отдельные единицы. Процессы получают только необходимые права - принцип минимальных привилегий.

## Проблема

Традиционная модель: обычный пользователь или root с неограниченными правами. Слишком грубо.

## Основные capabilities

CAP_NET_ADMIN - управление сетью
CAP_SYS_ADMIN - системное администрирование
CAP_DAC_OVERRIDE - обход проверок доступа
CAP_SETUID - изменение UID процесса
CAP_NET_BIND_SERVICE - bind на порты <1024

## Управление

```
getcap /usr/bin/nginx              # просмотр
setcap 'cap_net_bind_service=+ep' /usr/bin/nginx
```

## В контейнерах

Docker по умолчанию даёт только безопасный набор. Даже root внутри контейнера ограничен.
