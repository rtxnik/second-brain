# 31.04.01 Виртуальная машина

Виртуальная машина (VM) - программная эмуляция физического компьютера с собственной ОС. Работает изолированно от хоста и других VM, имеет выделенные ресурсы.

## Компоненты

**vCPU** - виртуальные процессорные ядра из физических ресурсов
**vRAM** - выделенная оперативная память из пула хоста
**Virtual Disk** - файл или блочное устройство, эмулирующее диск
**Network Interface** - виртуальная сетевая карта с MAC-адресом
**Guest OS** - операционная система внутри VM

## Состояния

| Состояние | Описание                                      |
| --------- | --------------------------------------------- |
| Running   | Активна, потребляет ресурсы                   |
| Stopped   | Выключена, ресурсы освобождены                |
| Suspended | Состояние сохранено в файл                    |
| Paused    | Выполнение приостановлено, память сохраняется |

## Операции

**Создание** - выделение ресурсов и установка ОС
**Клонирование** - копия существующей VM
**Snapshot** - сохранение состояния для отката
**Migration** - перенос между хостами (live/cold)
**Backup/Restore** - резервное копирование

## Размещение

**Локально** - VirtualBox, VMware Workstation
**Гипервизоры** - [[31.04.02 XCP-ng]], VMware vSphere, [[31.01.27 LXC|KVM]]
**Облако** - AWS EC2, Azure VM, Google Compute Engine

## VM vs Контейнеры

| Аспект   | VM                  | Контейнер         |
| -------- | ------------------- | ----------------- |
| Изоляция | Полная (гипервизор) | Процессная (ядро) |
| ОС       | Полная гостевая     | Общее ядро хоста  |
| Запуск   | Минуты              | Секунды           |
| Размер   | ГБ                  | МБ                |
| Overhead | Высокий             | Низкий            |
