# 32.01.08 Image

Docker Image - это неизменяемый шаблон для создания контейнеров. **Образ содержит все необходимое для запуска приложения: код, runtime, системные утилиты, библиотеки и настройки**.
Образы строятся из слоев (layers), каждый слой представляет изменения от предыдущего. Слои используют copy-on-write файловую систему и разделяются между образами для экономии дискового пространства.
Образы создаются из Dockerfile командой `docker build` или коммитом изменений из существующего контейнера. Каждая инструкция в Dockerfile создает новый слой в итоговом образе.

## Именование

Формат: `registry/namespace/repository:tag`

Примеры:

- `docker.io/library/nginx:latest`
- `myregistry.com/myapp:v1.2.3`

Если тег не указан, по умолчанию используется `latest`.

## Команды

```bash
docker pull nginx:alpine     # загрузка
docker push myapp:v1         # отправка
docker images                # список локальных
docker rmi myapp:v1          # удаление
```

## Базовые образы

| Образ   | Размер | Применение               |
| ------- | ------ | ------------------------ |
| scratch | 0 MB   | Минимальный, для Go/Rust |
| alpine  | ~5 MB  | Продакшен, минимализм    |
| ubuntu  | ~75 MB | Разработка, полный набор |

Alpine Linux популярен для продакшена из-за минимального размера.
