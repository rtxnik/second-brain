# 32.01.09 Network

Docker Network управляет сетевым взаимодействием между контейнерами и внешним миром. **Сети обеспечивают изоляцию трафика и позволяют контейнерам безопасно общаться друг с другом**.
По умолчанию Docker создает три сети: `bridge` (изолированная сеть для контейнеров), `host` (использование сети хоста) и `none` (отсутствие сетевого подключения).
Пользовательские bridge сети поддерживают Service Discovery — контейнеры могут обращаться друг к другу по именам вместо IP адресов. Это значительно упрощает конфигурацию многоконтейнерных приложений.

## Команды

```bash
docker network create mynet              # создание сети
docker network connect mynet container   # подключение контейнера
docker network disconnect mynet cont     # отключение
docker network ls                        # список сетей
docker network inspect mynet             # детальная информация
```

## Проброс портов

```bash
docker run -p 8080:80 nginx    # хост:контейнер
```

Позволяет получить доступ к контейнеру извне через порты хоста. Внутри пользовательской сети контейнеры общаются через внутренние порты без дополнительной настройки.
Overlay сети связывают контейнеры на разных Docker хостах и используются в кластерных решениях для распределенных приложений.
