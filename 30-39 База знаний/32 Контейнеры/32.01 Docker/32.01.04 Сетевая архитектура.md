# 32.01.04 Сетевая архитектура

Docker создает три сетевых драйвера по умолчанию: bridge, host и none. **Каждый драйвер решает разные задачи изоляции и доступности контейнеров**.
**Bridge** - сетевой интерфейс по умолчанию. Создает изолированную сеть, где контейнеры общаются между собой только по IP адресу. Контейнеры недоступны извне без явного проброса портов.
**Host** - контейнер использует сетевые настройки хоста напрямую, без собственного сетевого стека. Контейнер доступен по всем портам хоста и имеет максимальную производительность.
**None** - полная сетевая изоляция. Контейнер не имеет сетевых интерфейсов кроме loopback. Доступ возможен только через прямое подключение к среде выполнения.

## Пользовательские сети

**Custom Bridge** - аналогичен стандартному bridge, но с поддержкой Service Discovery - контейнеры могут обращаться друг к другу по DNS именам.
**Overlay** - связывают контейнеры на разных Docker хостах. Используются в кластерных решениях типа Docker Swarm.
**Macvlan** и **Ipvlan** - позволяют тонко настраивать IP адресацию. Macvlan дает каждому контейнеру уникальный MAC адрес, а Ipvlan использует MAC адрес хоста для всех контейнеров.
