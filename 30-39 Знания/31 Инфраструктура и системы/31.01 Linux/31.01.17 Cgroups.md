# 31.01.17 Cgroups

Control Groups - ограничение и учёт ресурсов для групп процессов.

## Подсистемы

**CPU** - лимиты процессорного времени, приоритеты (shares, quota).

**Memory** - лимиты RAM и swap.

**Block I/O** - пропускная способность дисков, IOPS.

**Devices** - контроль доступа к устройствам.

**Freezer** - приостановка/возобновление всех процессов группы.

## Версии

**v1** - отдельные иерархии для каждой подсистемы. Гибко, но сложно.

**v2** - единая иерархия. Проще в управлении.

## В контейнерах

Docker параметры транслируются в cgroups:

```
docker run --memory=512m --cpus=1.5 nginx
```

Kubernetes реализует QoS классы через cgroups.

## Управление

Файловая система /sys/fs/cgroup или systemd.
