# 31.01.03 Git коммиты

Стандартизированные префиксы для сообщений коммитов. Основаны на Conventional Commits. Обеспечивают понятную историю и автоматизацию.

## Основные типы

**feat:** — новая функциональность
```
feat: add user authentication
feat: implement dark theme
```

**fix:** — исправление багов
```
fix: resolve login redirect issue
fix: memory leak in data processing
```

**docs:** — изменения документации
```
docs: update README
docs: add API examples
```

**refactor:** — рефакторинг без изменения функциональности
```
refactor: extract validation logic
refactor: simplify database queries
```

**style:** — форматирование, не влияющее на логику
```
style: fix indentation
style: remove trailing whitespaces
```

## Дополнительные типы

**test:** — тесты. **chore:** — технические задачи, зависимости. **perf:** — производительность. **ci:** — CI/CD. **build:** — сборка. **revert:** — откат.

## Область изменений

Опционально в скобках:
```
feat(auth): add password reset
fix(api): handle null response
```

## Зачем это нужно

Автоматическая генерация changelog. Фильтрация коммитов по типам. Понятная история для команды. Интеграция с CI/CD.
