# 34.01.03 Архитектура Ansible

Ansible использует push-архитектуру без агентов. Control node подключается к managed hosts через SSH и выполняет модули Python.

## Control Node

- Управляющая машина с логикой Ansible
- Парсит playbook, подключается к хостам
- Отправляет модули и собирает результаты
- Требования: Python 3.8+, SSH клиент

## Managed Hosts

- Целевые системы для автоматизации
- Требуют только SSH сервер и Python
- Модули выполняются в изолированном окружении
- Возвращают JSON с результатами

## Процесс выполнения

```
1. Parsing    → чтение playbook и inventory
2. Connection → SSH соединения с хостами
3. Transfer   → отправка Python модулей
4. Execution  → выполнение и сбор фактов
5. Results    → получение JSON результатов
```

## Преимущества Agentless

- Нет ПО для обновления на тысячах серверов
- Стандартная SSH безопасность
- Простое развёртывание
