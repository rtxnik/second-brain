# 36.02 Микросервисы

Микросервисная архитектура — приложение как набор независимых сервисов. Каждый микросервис выполняет конкретную бизнес-функцию, имеет собственную БД и разрабатывается независимой командой.

Сервисы взаимодействуют через API (HTTP/REST, gRPC, async messaging).

## Характеристики

**Независимость развёртывания** — каждый сервис деплоится отдельно

**Database per Service** — собственное хранилище у каждого сервиса

**Технологическое разнообразие** — разные стеки для разных сервисов

**Команды владения** — каждый сервис принадлежит отдельной команде

**Бизнес-ориентированность** — сервисы организованы вокруг бизнес-функций

## Преимущества

| Преимущество | Почему важно |
|--------------|--------------|
| Независимое масштабирование | Каждый сервис отдельно |
| Технологическая свобода | Лучшие инструменты для каждой задачи |
| Отказоустойчивость | Падение одного не ломает всё |
| Быстрая разработка | Небольшие команды работают независимо |
| Простота понимания | Каждый сервис решает одну задачу |

## Недостатки

| Недостаток | Проблема |
|------------|----------|
| Сложность управления | Требует DevOps зрелости |
| Сетевые вызовы | Латентность и точки отказа |
| Eventual Consistency | Сложность поддержания консистентности |
| Мониторинг | Distributed tracing, централизованные логи |
| Инфраструктура | Overhead на каждый сервис |

## Технологический стек

| Компонент | Назначение |
|-----------|------------|
| Docker | Упаковка сервисов |
| Kubernetes | Оркестрация контейнеров |
| API Gateway | Точка входа и маршрутизация |
| Service Discovery | Автообнаружение сервисов |
| Config Management | Централизованная конфигурация |

## Паттерны

**API Gateway** — единая точка входа для клиентов

**Circuit Breaker** — защита от каскадных сбоев

**Saga Pattern** — распределённые транзакции

**Event Sourcing** — хранение изменений как событий

## Когда выбирать

- Большие команды (20+ разработчиков)
- Сложные приложения с множественными доменами
- Высокие требования к масштабируемости
- Зрелые DevOps практики
