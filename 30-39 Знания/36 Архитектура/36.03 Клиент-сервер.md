# 36.03 Клиент-сервер

Клиент-серверная архитектура - распределённая модель взаимодействия, где клиенты запрашивают услуги у серверов по сети. Разделяет потребителя услуг (клиент) и поставщика (сервер).

## Компоненты

**Клиент** - инициирует запросы, отображает результаты

**Сервер** - обрабатывает запросы, выполняет логику, управляет данными

**Протокол** - формат обмена (HTTP, TCP, WebSocket)

**Middleware** - маршрутизация, безопасность, кеширование

## Типы клиентов

| Тип          | Логика                      | Пример                    |
| ------------ | --------------------------- | ------------------------- |
| Thin Client  | Минимальная, всё на сервере | Веб-браузер               |
| Thick Client | Существенная на клиенте     | Desktop приложения        |
| Rich Client  | Сбалансированно             | SPA, мобильные приложения |

## Типы серверов

| Тип                | Функция            | Пример               |
| ------------------ | ------------------ | -------------------- |
| Web Server         | HTTP запросы       | Nginx, Apache        |
| Application Server | Бизнес-логика      | Node.js, Spring Boot |
| Database Server    | Управление данными | PostgreSQL, MongoDB  |
| File Server        | Доступ к файлам    | FTP, SMB             |

## Преимущества

- Централизованное управление данными и логикой
- Масштабируемость - серверы и клиенты независимо
- Безопасность - контроль на серверной стороне
- Обновления - изменения доступны всем клиентам

## Недостатки

- Единая точка отказа - падение сервера блокирует всех
- Сетевая зависимость - требуется стабильное соединение
- Латентность - сетевые задержки
- Нагрузка - все запросы централизованно

## Современные реализации

Веб-приложения - браузер ↔ веб-сервер

REST API - HTTP-клиенты ↔ REST серверы

Docker - клиент ↔ демон

БД - приложение ↔ сервер БД
