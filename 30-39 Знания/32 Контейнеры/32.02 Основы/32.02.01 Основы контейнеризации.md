# 32.02.01 Основы контейнеризации

Контейнеризация — это технология упаковки приложений с их зависимостями в изолированные переносимые контейнеры. **Контейнеры используют общее ядро операционной системы, что делает их более легковесными по сравнению с виртуальными машинами**.
Основная идея контейнеризации — создать стандартизированную единицу развертывания, которая работает одинаково в любой среде: на ноутбуке разработчика, тестовом сервере или в продакшене.

## Ключевые принципы

**Изоляция процессов** — каждый контейнер работает в собственном изолированном пространстве
**Портативность** — "собрал один раз, запускается везде"
**Легковесность** — разделяемое ядро ОС снижает потребление ресурсов
**Масштабируемость** — быстрый запуск и остановка контейнеров
**Консистентность** — одинаковое поведение во всех окружениях

## Технологии ядра Linux

Контейнеризация построена на функциях ядра Linux:

**[[31.01.16 Namespaces]]** — изолируют ресурсы процессов (сеть, файловая система, процессы)
**[[31.01.17 Cgroups]]** — ограничивают потребление ресурсов (CPU, память, дисковый I/O)
**[[31.01.15 Capabilities]]** — обеспечивают гранулярные привилегии без полных прав root

## Сравнение с виртуализацией

| Характеристика | Контейнеры | Виртуальные машины |
|----------------|------------|-------------------|
| **Ресурсы** | Низкое потребление | Высокое потребление |
| **Запуск** | Секунды | Минуты |
| **Изоляция** | Уровень процессов | Аппаратная виртуализация |
| **Портативность** | Очень высокая | Средняя |
| **Безопасность** | Разделяемое ядро | Полная изоляция |
| **Размер** | МБ | ГБ |

## Архитектурные модели

Контейнеризация идеально подходит для [[36.02 Микросервисы|микросервисной архитектуры]], где каждый сервис работает в отдельном контейнере. Это обеспечивает независимость развертывания, масштабирования и обновления компонентов.

## Основные реализации

**[[32.01.00 Docker]]** — самая популярная платформа контейнеризации приложений
**[[31.01.27 LXC]]** — системные контейнеры с полной ОС
**[[31.01.28 LXD]]** — современный менеджер LXC контейнеров с REST API
**Podman** — daemon-less альтернатива Docker

## Применение в DevOps

Контейнеризация стала основой современных DevOps практик, обеспечивая:

- **CI/CD пайплайны** — консистентная среда от разработки до продакшена
- **Infrastructure as Code** — декларативное описание инфраструктуры
- **Мониторинг и логирование** — стандартизированные способы сбора метрик
