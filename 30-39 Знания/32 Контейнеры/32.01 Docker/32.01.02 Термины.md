# 32.01.02 Термины

## Базовые понятия контейнеризации

**Контейнер** - изолированная среда выполнения приложения с необходимыми зависимостями.
**Контейнеризация** - технология упаковки приложений с зависимостями в портативные контейнеры.
**Оркестрация** - автоматизированное развертывание, масштабирование и управление контейнерами.
**Микросервисная архитектура** - подход разработки с использованием множества небольших независимых сервисов.

## Архитектура Docker

**Docker Engine** - полная система Docker (клиент + демон + API).
**Docker Client** - интерфейс командной строки для взаимодействия с Docker.
**Docker Daemon** - фоновый процесс, управляющий контейнерами и образами.
**REST API** - программный интерфейс для взаимодействия клиента с демоном.

## Образы и контейнеры

**Image** - неизменяемый шаблон для создания контейнеров.
**Container** - запущенный экземпляр образа.
**Dockerfile** - текстовый файл с инструкциями для сборки образа.
**Layer** - слой файловой системы в образе Docker.
**Base Image** - исходный образ, на основе которого строится новый.
**Tag** - метка для версионирования образов (`nginx:latest`, `ubuntu:20.04`).
**Multi-stage Build** - техника сборки образов с несколькими этапами для уменьшения размера.

## Хранилища и реестры

**Registry** - сервис для хранения и распространения образов.
**Docker Hub** - публичный облачный реестр Docker образов.
**Private Registry** - приватный реестр для внутренних образов компании.
**Pull** - загрузка образа из реестра.
**Push** - загрузка образа в реестр.
**Repository** - коллекция связанных образов с разными тегами.

## Управление данными

**Volume** - управляемое Docker постоянное хранилище.
**Bind Mount** - прямое монтирование директорий хоста.
**tmpfs Mount** - временное хранилище в оперативной памяти.
**Anonymous Volume** - том, созданный автоматически без имени.
**Named Volume** - том с явно заданным именем.

## Сетевое взаимодействие

**Docker Network** - виртуальная сеть для контейнеров.
**Bridge Network** - сетевой драйвер по умолчанию для изоляции контейнеров.
**Host Network** - использование сетевого стека хоста контейнером.
**Overlay Network** - сеть для связи контейнеров на разных хостах.
**Service Discovery** - автоматическое обнаружение сервисов по имени.
**Port Mapping** - проброс портов контейнера на хост (`-p 8080:80`).

## Оркестрация и многоконтейнерные приложения

**Docker Compose** - инструмент для описания многоконтейнерных приложений.
**docker-compose.yml** - файл конфигурации для Compose.
**Service** - описание желаемого состояния группы контейнеров.
**Stack** - группа взаимосвязанных сервисов.
**Scale** - изменение количества экземпляров сервиса.

## Операционные команды

**Build** - создание образа из Dockerfile.
**Run** - запуск нового контейнера.
**Start/Stop** - управление жизненным циклом контейнера.
**Exec** - выполнение команды в работающем контейнере.
**Logs** - просмотр логов контейнера.
**Inspect** - детальная информация об объекте Docker.
**Prune** - очистка неиспользуемых объектов Docker.

## Специализированные концепции

**Health Check** - проверка состояния контейнера.
**Init System** - процесс инициализации внутри контейнера.
**Secrets** - безопасное управление конфиденциальными данными.
**Config** - управление конфигурационными файлами.
