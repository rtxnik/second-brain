# 32.01.07 Container

Docker Container — это запущенный экземпляр образа. **Контейнер представляет изолированную среду выполнения со всеми необходимыми зависимостями для работы приложения**.

Контейнер создается на основе образа и добавляет сверху записываемый слой для изменений во время выполнения. Все изменения в контейнере — новые файлы, модификации существующих — сохраняются в этом слое и исчезают при удалении контейнера.

Изоляция обеспечивается технологиями ядра Linux: [[31.01.16 Namespaces]] изолируют процессы, сеть и файловую систему, а [[31.01.17 Cgroups]] ограничивают использование CPU, памяти и дискового I/O.

Основные операции включают `docker run` (создание и запуск), `docker start/stop` (управление состоянием), `docker exec` (выполнение команд внутри), `docker logs` (просмотр логов), `docker rm` (удаление).

Контейнеры могут быть stateless (без сохранения данных) или stateful (с использованием томов для постоянного хранения). Stateless подход предпочтителен для [[36.02 Микросервисы|микросервисной архитектуры]].
