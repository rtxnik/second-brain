# 32.01.11 Bind Mounts

Bind Mounts монтируют файл или директорию с хоста напрямую в контейнер. **Изменения в примонтированных файлах мгновенно отражаются в контейнере и наоборот, что идеально подходит для разработки**.
В отличие от volumes, bind mounts используют абсолютный путь к файлу или директории на хосте. Это дает полный контроль над расположением данных, но снижает портативность между разными системами.

## Монтирование

```bash
# Короткий синтаксис
docker run -v /host/path:/container/path myapp

# Длинный синтаксис
docker run --mount type=bind,source=/host/path,target=/container/path myapp

# Только чтение
docker run --mount type=bind,source=/host/path,target=/data,readonly myapp
```

## Сравнение с Volumes

| Аспект        | Bind Mounts      | Volumes                    |
| ------------- | ---------------- | -------------------------- |
| Расположение  | Любой путь хоста | `/var/lib/docker/volumes/` |
| Управление    | Пользователь     | Docker                     |
| Портативность | Низкая           | Высокая                    |
| Безопасность  | Ниже             | Выше                       |

## Применение

- **Разработка** - монтирование исходного кода для live reload
- **Конфигурация** - доступ к конфигурационным файлам хоста
- **Логирование** - запись в системные директории

Bind mounts менее безопасны чем volumes, так как контейнер получает прямой доступ к файловой системе хоста. Контейнер может изменять критически важные системные файлы при наличии соответствующих прав.
Права доступа определяются пользователем внутри контейнера и правами на файлы хоста. Это может вызывать проблемы с доступом при несовпадении UID/GID.
