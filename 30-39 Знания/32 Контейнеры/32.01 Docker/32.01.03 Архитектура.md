# 32.01.03 Архитектура

Docker использует клиент-серверную архитектуру. **Клиент общается с демоном Docker через REST API для управления контейнерами, образами, сетями и томами**.
Docker Client - это интерфейс командной строки (`docker run`, `docker build`), который отправляет команды демону. Клиент и демон могут работать как на одной машине, так и на разных хостах.
Docker Daemon (`dockerd`) - это фоновый сервис, который выполняет всю тяжелую работу: создает и управляет контейнерами, собирает образы, управляет сетями и томами. Демон также взаимодействует с реестрами для загрузки и отправки образов.
REST API предоставляет программный интерфейс для взаимодействия с демоном. Любое приложение может использовать API напрямую, минуя CLI.
Docker демон работает только на Linux, так как зависит от функций ядра Linux - [[31.01.16 Namespaces]] и [[31.01.17 Cgroups]]. На Windows и macOS Docker запускается внутри виртуальной машины Linux.

## Схема архитектуры

```
┌─────────────┐     REST API     ┌─────────────┐
│   Docker    │ ───────────────► │   Docker    │
│   Client    │                  │   Daemon    │
│   (CLI)     │                  │  (dockerd)  │
└─────────────┘                  └──────┬──────┘
                                        │
                          ┌─────────────┼─────────────┐
                          ▼             ▼             ▼
                    ┌──────────┐  ┌──────────┐  ┌──────────┐
                    │Container │  │  Image   │  │ Network  │
                    └──────────┘  └──────────┘  └──────────┘
```
