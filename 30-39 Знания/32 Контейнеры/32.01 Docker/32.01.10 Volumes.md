# 32.01.10 Volumes

Docker Volumes - это управляемые Docker постоянные хранилища данных. **Volumes обеспечивают сохранность данных независимо от жизненного цикла контейнеров и позволяют разделять данные между несколькими контейнерами**.
При создании тома Docker размещает его в специальной директории хоста (`/var/lib/docker/volumes/`) под управлением демона. Это обеспечивает изоляцию от прямого вмешательства пользователей и дополнительную безопасность.
Volumes поддерживают различные драйверы хранения: локальные диски, сетевые файловые системы (NFS), облачные хранилища (AWS EBS, Azure Disk). Это позволяет легко мигрировать данные между хостами.

## Команды

```bash
docker volume create myvolume   # создание
docker volume ls                # список
docker volume inspect myvolume  # детальная информация
docker volume rm myvolume       # удаление
docker volume prune             # очистка неиспользуемых
```

## Монтирование

```bash
# Короткий синтаксис
docker run -v myvolume:/data myapp

# Длинный синтаксис
docker run --mount source=myvolume,target=/data myapp
```

## Типы томов

**Именованные (Named)** - создаются явно и проще в управлении
**Анонимные (Anonymous)** - создаются автоматически при использовании `VOLUME` в Dockerfile
Volumes остаются после удаления контейнера и должны удаляться отдельно. Это предотвращает случайную потерю важных данных.
