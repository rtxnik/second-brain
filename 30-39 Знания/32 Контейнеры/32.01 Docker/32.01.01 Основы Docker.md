# 32.01.01 Основы Docker

Docker — платформа контейнеризации приложений с открытым исходным кодом. **Основная идея: упаковать приложение со всеми зависимостями в переносимый контейнер, который одинаково работает в любой среде**.
Контейнеры используют общее ядро хоста и потребляют значительно меньше ресурсов чем виртуальные машины. Запуск нового контейнера занимает секунды вместо минут.
Docker построен на технологиях ядра Linux — [[31.01.16 Namespaces]] для изоляции и [[31.01.17 Cgroups]] для ограничения ресурсов.
Базовый процесс работы: разработчик создает [[32.01.13 Dockerfile]], Docker собирает из него [[32.01.08 Image]], который запускается как [[32.01.07 Container]].
Основные компоненты: [[32.01.05 Client]] (CLI), [[32.01.06 Daemon]] (сервис управления) и [[32.01.12 Registry]] (хранилище образов).
