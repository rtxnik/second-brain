## Обзор

Операционные процедуры для поддержания работоспособности homelab:

- Мониторинг питания и UPS
- Резервное копирование
- Обновления систем
- Disaster recovery

---

## Критические процедуры

### При отключении питания

```
1. UPS → NUT Master (TrueNAS) → детектит low battery
2. NUT Master → оповещает NUT Slaves
3. Slaves (Proxmox ноды) → graceful shutdown VMs
4. Slaves → shutdown хостов
5. Master → shutdown TrueNAS (последний)
```

### При сбое ноды Proxmox

```
1. Ceph детектирует недоступность OSD
2. HA manager мигрирует VMs на другие ноды
3. Ceph начинает recovery (если >1 OSD down)
4. Администратор: диагностика и восстановление
```

## Мониторинг

| Что          | Как           | Алерт        |
| ------------ | ------------- | ------------ |
| Proxmox ноды | Prometheus    | Down > 1 min |
| Ceph health  | ceph status   | WARN/ERR     |
| UPS battery  | NUT           | < 50%        |
| Disk space   | Node exporter | > 80%        |
| K8s nodes    | Prometheus    | NotReady     |

## Maintenance Windows

```
Рекомендуемое время: воскресенье, 03:00-06:00
Уведомление: за 24 часа

Процедура:
1. Уведомить (если есть пользователи)
2. Создать snapshot/backup
3. Выполнить обслуживание
4. Проверить работоспособность
5. Документировать изменения
```
